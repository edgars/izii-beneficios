## Visão geral

Esta página disponibiliza a **coleção Postman** usada para chamar o Orquestrador
no modelo **legado** (endpoints `POST` em `/api/orquestrador/...`), alinhado ao
[Manual de Orientação Técnica](/manual-orquestrador.pdf) e à página
[Acesso Direto ao Orquestrador Izzi](/docs/orquestrador-izzi).

Para integrações novas, prefira a [Referência REST](/docs/referencia-rest) e a
spec em [`/izzi-rest`](/izzi-rest).

## O que a coleção inclui

| Pasta / pedido | Método | URL (variável `{{baseURL}}`) |
|---|---|---|
| **Realizar Integração → Inclusão** | `POST` | `{{baseURL}}api/orquestrador/realizar-integracao` |
| **Auth - Gerar Token** | `POST` | `{{baseURL}}api/orquestrador/gerar-jwt-para-acesso` |

O pedido **Auth - Gerar Token** inclui um script de teste que grava o token na
variável de coleção `authorization_token` a partir de `response.token`, para
reutilizar no `Authorization: Bearer {{authorization_token}}` do pedido de
integração.

## Variáveis da coleção

Configure no Postman (**Collection → Variables**):

| Variável | Uso |
|---|---|
| `baseURL` | Prefixo do host (ex.: `https://seu-host:44304/` — inclua barra final se os paths da coleção o assumirem). |
| `clientId` | Credencial do cliente (contrato legado do gerador de JWT). |
| `clientSecret` | Segredo do cliente. |
| `tenantId` | Identificador do inquilino. |
| `authorization_token` | Preenchido automaticamente após correr **Auth - Gerar Token** (script de teste). |

> **Nota:** A API REST documentada em `/izzi-rest` usa `consumerKey`,
> `consumerSecret` e `tenantId` em `POST /v1/auth/token`. Esta coleção legada
> usa `clientId` / `clientSecret` no corpo do `gerar-jwt-para-acesso` — alinhe
> com o ambiente que estiver a testar (mock, gateway ou backend real).

## Passos rápidos no Postman

1. **Importar** o JSON descarregado acima.
2. Ajustar **`baseURL`** (e credenciais) às variáveis da coleção.
3. Executar **Auth - Gerar Token** e confirmar que `authorization_token` foi
   atualizada (aba *Variables* ou consola de testes).
4. Executar **Inclusão** (ou duplicar o pedido para outros movimentos, se
   aplicável).
